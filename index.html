📖 Section 2

Terminologie monday.com

Les termes essentiels expliqués simplement. Chaque concept en une phrase.

🏢
Espace de travail (Workspace)
Le dossier principal qui regroupe tous vos tableaux par thème ou département.
Exemple : Espace de travail « Marketing » contient tous les projets marketing.
📊
Tableau (Board)
Le tableau principal où vit votre projet. C'est votre espace de travail quotidien.
Exemple : Tableau « Lancement Produit 2026 »
📁
Groupe (Group)
Une section colorée dans un tableau pour organiser les tâches par catégorie.
Exemple : Groupes « Phase 1 », « Phase 2 », « Phase 3 »
✅
Élément / Tâche (Item)
Une ligne dans votre tableau = une tâche, une action ou un livrable.
Exemple : « Créer le visuel Instagram »
🔗
Sous-tâche (Subitem)
Une tâche imbriquée sous un item pour détailler les étapes.
Exemple : Sous-tâche « Valider le texte » sous « Créer le visuel »
📐
Colonne (Column)
Les informations associées à chaque tâche : statut, personne, date, etc.
Exemple : Colonnes « Statut », « Responsable », « Échéance »
🚦
Statut (Status)
Un indicateur coloré qui montre l'avancement d'une tâche.
Exemple : Terminé En cours Bloqué
👤
Personnes (People)
La colonne pour assigner un ou plusieurs responsables à une tâche.
Exemple : Assigner « Aymen » à la tâche design
📅
Chronologie (Timeline)
Une barre visuelle montrant la date de début et de fin d'une tâche.
Exemple : Du 1er au 15 juin 2026
📈
Tableau de bord (Dashboard)
Une page avec des graphiques et KPIs pour suivre vos projets en un coup d'œil.
Exemple : Tableau de bord « Suivi Trimestriel »
⚡
Automatisation (Automation)
Une règle « Si… alors… » qui exécute des actions automatiquement.
Exemple : « Quand statut = Terminé → notifier le manager »
🔌
Intégration (Integration)
Connecter monday.com à d'autres outils (Gmail, Slack, Teams…).
Exemple : Recevoir un email quand une tâche est assignée
💬
Commentaire (Update)
Un fil de discussion attaché à chaque tâche pour communiquer en contexte.
Exemple : « @Mouad peux-tu valider ce paramétrage ERP ? »
👁️
Vue (View)
Différentes façons de visualiser vos données : tableau, Kanban, calendrier…
Exemple : Passer de la vue Tableau à la vue Kanban
🔒
Types de Boards
Main Board = visible par tous · Private = sur invitation · Shareable = accessible aux invités externes
Conseil : Commencez toujours par un Main Board

🚀 Section 4

Créer votre premier projet

Suivez ces étapes pour mettre en place un vrai projet. Exemple : « Lancement Produit 2026 ».

1

Créer un Espace de travail

Cliquez sur « + » dans la barre latérale gauche, puis « Nouvel espace de travail ». Nommez-le selon votre département ou projet.

📍 Barre latérale → « + » → Espace de travail
➕ Nouveau Workspace
Nom du workspace :
Mon Entreprise
2

Créer un Tableau (Board)

Dans votre espace de travail, cliquez « + Ajouter » → « Nouveau Tableau ». Choisissez un nom clair et descriptif.

📍 Espace de travail → « + Ajouter » → Tableau
📊 Lancement Produit 2026
Type : Main Board (visible par toute l'équipe)
3

Ajouter des Groupes

Les groupes organisent vos tâches par phase, mois ou catégorie. Cliquez « + Ajouter un groupe ».

📍 En haut du board → « + Groupe »
📁 Phase 1 — Préparation
📁 Phase 2 — Exécution
📁 Phase 3 — Suivi
4

Ajouter des tâches (Items)

Cliquez « + Ajouter un élément » en bas de chaque groupe. Chaque ligne = une tâche à réaliser.

📍 Bas du groupe → « + Élément »
Phase 1 — Préparation
Analyse marché Terminé NI 1-10 juin
Brief créatif En cours MC 5-12 juin
Validation budget Bloqué SA 8-15 juin
5

Assigner et planifier

Cliquez sur la colonne Personne pour assigner un responsable, et sur Chronologie pour définir les dates.

📍 Cliquer sur la cellule → Choisir
👤 Responsable :
IM Imed
📅 Période :
1 juin → 15 juin

Les meilleures vues pour débuter

Changez de vue pour voir vos données différemment. Même données, perspectives différentes.

📊
Vue Tableau (Table)
La vue par défaut. Comme un tableur amélioré avec colonnes personnalisables.
⭐ Débutant
✅ Idéal pour : gestion quotidienne des tâches
🗂️
Vue Kanban
Des cartes organisées en colonnes (À faire → En cours → Terminé).
⭐ Débutant
✅ Idéal pour : suivre l'avancement visuel
📅
Vue Calendrier
Vos tâches affichées sur un calendrier mensuel ou hebdomadaire.
⭐ Débutant
✅ Idéal pour : planifier et voir les deadlines
📏
Vue Chronologie (Timeline)
Barres horizontales montrant la durée de chaque tâche dans le temps.
📙 Intermédiaire
✅ Idéal pour : planification de projets complexes
📉
Vue Gantt
Vue avancée avec dépendances entre les tâches et chemin critique.
🔴 Avancé
✅ Idéal pour : gestion de projet avancée
⚡ Section 6

Les automatisations simplifiées

Les automatisations font le travail répétitif à votre place. Aucun code requis !

💡 Une automatisation = une règle simple : « QUAND quelque chose se passe → ALORS faire une action »
🔔 QUAND
Le statut change
→
✉️ ALORS
Notifier le manager
📅 QUAND
La date arrive
→
🔴 ALORS
Changer le statut en « Urgent »
✅ QUAND
Toutes les sous-tâches = Terminé
→
🎉 ALORS
Marquer l'item comme Terminé

🏆 Top 5 automatisations pour débuter

1️⃣
Notification de statut
Quand le statut change → notifier la personne assignée
2️⃣
Rappel de deadline
Quand la date arrive → envoyer un rappel
3️⃣
Déplacer un item
Quand statut = Terminé → déplacer vers le groupe « Archivé »
4️⃣
Assignation auto
Quand un élément est créé → assigner automatiquement à Mohammed
5️⃣
Création de sous-tâches
Quand un item est créé → créer les sous-tâches standard
📈 Section 7

Les Dashboards simplifiés

Un dashboard = une page de suivi avec des graphiques pour tout voir en un coup d'œil.

👔 Pour les managers : Le dashboard est votre meilleur outil. Sans ouvrir chaque board, vous voyez l'état de tous vos projets.

🧩 Les widgets essentiels

🔢
Widget Chiffres
Affiche un KPI clé : nombre de tâches, % terminé, budget restant.
📊
Widget Graphique
Camembert ou barres pour visualiser la répartition des statuts.
📅
Widget Deadlines
Liste les tâches en retard ou arrivant bientôt à échéance.
👥
Widget Charge d'équipe
Montre qui est surchargé et qui a de la disponibilité.

⚠️ Section 9

Erreurs fréquentes des débutants

Évitez ces pièges courants pour une expérience réussie dès le premier jour.

❌ À ÉVITER
🚫 Créer un board par tâche (trop de boards)
🚫 Noms vagues : « Board 1 », « Projet X »
🚫 15 statuts différents par board
🚫 Automatisations complexes dès le début
🚫 Ne pas assigner de responsable
🚫 Tout mettre dans un seul groupe
🚫 Ignorer les permissions des boards
✅ BONNE PRATIQUE
✅ Un board par projet, des groupes par phase
✅ Noms clairs : « Marketing Q2 2026 »
✅ 4-5 statuts max : À faire, En cours, Review, Terminé
✅ Commencer par 2-3 automatisations simples
✅ Toujours assigner un responsable unique
✅ Utiliser des groupes pour organiser
✅ Boards privés pour les données sensibles (RH, Finance)
⚡ Règle d'or : Commencez simple. Vous pourrez toujours ajouter de la complexité plus tard. Un board clair et simple sera toujours plus efficace qu'un board complexe et confus.
